### 2026-03-27 — Модель «Пространство» (Space) и переход от привязки к организации

**Что сделано:** Введена новая сущность «Пространство» (Space), которая заменяет прежнюю привязку пользователя к организации (Company). ### 2026-03-27 — Расширение модели Person (новые поля)

**Что сделано:** В карточку физического лица добавлены новые поля: пол, адрес, уровень образования, паспортные данные, телефон. СНИЛС стал необязательным (раньше был уникальным обязательным полем). **Технические детали:** Новые поля в модели `Person`: `gender`, `address`, `edu_level`, `passport`, `phone`. Поле `snils` изменено на `blank=True, default=''`. Поле `code` расширено до `max_length=20`. Миграции: `0012`, `0013`. Обновлены fieldsets в `PersonAdmin`.

### 2026-03-27 — Новый механизм автосоздания аккаунтов

**Что сделано:** Полностью переписан механизм автоматического создания учётных записей при добавлении физического лица. Раньше логин формировался из цифр СНИЛС, пароль — первые 6 цифр СНИЛС. Теперь: логин = ID записи (число), пароль = автоматически сгенерированный 6-значный код. Код генерируется случайно при создании, если не задан вручную.

**Как проверить:**
1. Создать нового человека в системе (Физические лица → Создать).
2. После сохранения у записи автоматически появится 6-значный код в поле «Код доступа».
3. Учётная запись создастся автоматически — логин будет равен ID записи, пароль — сгенерированному коду.

**Технические детали:** Переписан `courses/signals.py`: сигнал `post_save` на `Person` генерирует код через `random.randint` и вызывает `person.create_user_account()`. Подключение сигналов перенесено в `apps.py` через метод `ready()`.
